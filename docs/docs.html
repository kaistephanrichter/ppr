Developer Documentation

Everything you need to build, run, and contribute to PPR — a native iOS client for paperless-ngx.

Overview

PPR is a SwiftUI iOS app that connects to a self-hosted paperless-ngx instance via its REST API. The app targets a single user (or small household): you supply the server URL and an API token — there is no bundled service.

The app focuses on ingest, find, read, and status. It intentionally does not replicate the full paperless-ngx web UI.

Requirements

  • Xcode 16+
  • iOS 18.0+ deployment target
  • XcodeGen — brew install xcodegen
  • Fastlane (optional, for screenshots) — brew install fastlane
  • A running paperless-ngx instance reachable from your device

Build from Source

1. Clone the repository

git clone https://github.com/kaistephanrichter/ppr.git
cd ppr

2. Generate the Xcode project

The project is not committed — it is generated from project.yml using XcodeGen.

xcodegen generate

3. Open in Xcode

open PPR.xcodeproj

4. Configure signing

Update the bundle identifier and team in Xcode's Signing & Capabilities settings to match your Apple Developer account.

Note: After any changes to project.yml, re-run xcodegen generate to regenerate the project file.

API Layer

All network calls go through the PaperlessAPI module in Sources/PPR/API/. It handles:

  • Base URL + Authorization: Token <token> header injection
  • Paginated responses (count, next, previous, results)
  • Codable models for all response types
  • Structured error types for network and server errors

The paperless-ngx OpenAPI schema (GET /api/schema/) is the source of truth for request/response shapes.

Key endpoints used

  • GET /api/documents/ — document list with search & filter
  • POST /api/documents/post_document/ — upload
  • GET /api/documents/{id}/download/ — PDF download
  • GET /api/tags/, /api/document_types/, /api/correspondents/ — metadata
  • GET /api/status/ — server health

Configuration & Security

Server URL and API token are stored in the iOS Keychain via Sources/PPR/Configuration/. They are never logged or stored in plain text.

Important: GET /api/profile/ echoes the auth_token in its response — never log this endpoint's output.

HTTP (non-TLS) connections to local network servers are explicitly allowed via NSAllowsLocalNetworking in Info.plist, which is required for typical home server setups.

XcodeGen

The Xcode project (PPR.xcodeproj) is generated from project.yml and is not committed to the repository. This keeps the repo clean and avoids merge conflicts in Xcode project files.

# Regenerate after any changes to project.yml
xcodegen generate

Key things configured in project.yml: targets, bundle IDs, deployment target, build settings, entitlements, and Info.plist values.

App Store Screenshots

Screenshots are generated automatically using Fastlane Snapshot and the UI test target PPRUITests.

# Generate screenshots for all devices and languages
fastlane screenshots

Output is saved to fastlane/screenshots/, organized by language and device type.

Localization

PPR is fully translated in 7 languages:

  • German (de)
  • English (en)
  • French (fr)
  • Spanish (es)
  • Italian (it)
  • Dutch (nl)
  • Polish (pl)

String catalogs are located in Sources/PPR/Resources/. All user-facing strings must be localized using SwiftUI's standard localization APIs.
